
# 작가 전용 스토리 빌딩 웹 서비스 (Story Planner Web)

## 1. 프로젝트 개요 (Overview)

- **서비스명:** 스토리 플래너 Web (Story Planner Web)
    
- **타겟 유저:** 장르소설(웹소설, 판타지, SF, 로맨스 등) 및 일반 소설 작가, 시나리오 라이터
    
- **기획 목적:** PDF 형태의 정형화된 스토리 빌딩 템플릿의 장점과 자유로운 웹 스크랩(포스트잇 방식)의 장점을 결합하여, 집필과 자료조사를 실시간으로 넘나드는 통합 작업 환경 제공.
    
- **디자인 및 개발 방향성:**
    
    - **디자인:** 장시간 텍스트를 보아도 눈이 피로하지 않은 다크 모드 지원, 캔버스 영역의 자유로운 Drag & Drop 인터페이스 최적화.
        
    - **개발:** 유기적인 데이터 연동(캐릭터 설정 변동 시 회차 플래너 실시간 반영), 비정형 데이터(웹 클리핑)의 파싱 및 태깅 시스템 구축.
        

## 2. 핵심 메뉴 및 상세 기능 요구사항 (Features Specification)

### Tab 1. 마스터 시놉시스 (Synopsis Master)

작품의 거시적 정보와 핵심 뼈대를 정의하는 정형 폼(Form) 스페이스.

- **UI/UX:** 고정형 레이아웃, 줄글 입력 시 가독성을 높인 에디터 컴포넌트 사용.
    
- **상세 기능:**
    
    - **기본 메타데이터:** 작품명, 장르, 예상 분량 입력 필드.
        
    - **핵심 요약:** 로그라인(한 줄 요약), 기획의도 텍스트 필드.
        
    - **5단 플롯 요약:** **[발단 - 전개 - 위기 - 절정 - 결말]** 구조를 시각적으로 분할한 카드형 대시보드(각 영역 텍스트 입력 및 수정 가능).
        

### Tab 2. 스크랩 보드 (Scrap Pad) - _핵심 추가 기능_

자료조사 링크 및 메모를 포스트잇 형태로 자유롭게 배치하고 찾아 쓰는 무한 캔버스 공간. 원할 경우 표 데이터베이스로 확인 가능.

- **UI/UX:** 미로(Miro)나 피그마(Figma) 같은 무한 캔버스 형태. Drag & Drop 인터페이스 필수.
    
- **상세 기능:**
    
    - **포스트잇 링크 스크랩:** URL 입력 시 Open Graph(OG) 태그를 파싱하여 타이틀, 대표 이미지, 핵심 설명을 자동으로 추출해 카드로 생성.
        
    - **시각적 분류:** 카드 배경색 선택 기능(기본 5종 컬러칩 제공), 중요 카드 핀 고정(Pin) 기능.
        
    - **스마트 검색 및 필터링:** * 상단 인스턴트 키워드 검색창: 타이핑 시 실시간으로 매칭되지 않는 카드는 블러(Blur) 처리 또는 페이드아웃.
        
        - 태그(#) 필터 시스템: 태그 클릭 시 해당 태그를 가진 카드만 모아보기.
            

### Tab 3. 다차원 세계관 위키 (World-Building Wiki)

소설의 배경 설정을 구조화하여 보관하는 아카이빙 탭.

- **UI/UX:** 노션(Notion)의 토글(Toggle) 뷰 또는 위키(Wiki) 구조. 카테고리별 아코디언 메뉴 레이아웃.
    
- **상세 기능:**
    
    - **시공간 영역:** 주요 장소(공간), 시대 및 연호(시간).
        
    - **사회/인프라 영역:** 환경, 언어, 역사, 정치 체제, 경제 구조, 사회적 관습.
        
    - **정신/제약 영역:** 종교, 신화, 규율, 그리고 서사의 핵심 트리거가 될 금기(Taboo) 설정 필드.
        

### Tab 4. 캐릭터 데이터베이스 (Character DB)

인물 프로필 설정 및 인과적 서사 추적 모듈.

- **UI/UX:** 그리드 형태의 캐릭터 리스트(퀵 뷰) + 상세 클릭 시 슬라이드 아웃 패널(Detail Panel).
    
- **상세 기능:**
    
    - **프로필 상세 모델링:** 외적 요소(이름, 나이, 성별, 외모 묘사 등) 및 내적 요소(장단점, 가치관, 트라우마, 결핍, 내재적 욕구/목적, 시그니처 대사) 구분 저장.
        
    - **관계도 매핑 (Relationship Linker):** 캐릭터 카드 간 선(Line)을 연결하고 '우호/적대/동맹' 등의 관계 텍스트를 오버레이하는 컴포넌트.
        
    - **인물 변화표 (Arc Matrix):** '고정값(욕망, 목적)'과 '회차별 변동값(과제 $\rightarrow$ 목표 $\rightarrow$ 감정 변화 $\rightarrow$ 행동 $\rightarrow$ 결과)'을 매트릭스로 매핑.
        
    - **인물 등장표 (Presence Grid):** X축(회차 1~50화+), Y축(캐릭터 목록)으로 구성된 체크박스 그리드. 특정 회차에 캐릭터 등장 여부(On/Off) 시각화.
        

### Tab 5. 타임라인 플래너 & 씬 디자이너 (Timeline & Scene)

스토리 전개 흐름 제어 및 개별 사건의 인과관계 설계.

- **UI/UX:** 캘린더/간트차트 형태의 대시보드 뷰 + 트리 구조의 사건 링크 뷰.
    
- **상세 기능:**
    
    - **스토리 진행률 매핑:** 전체 분량 대비 현재 서사의 위치(10%~100%)를 시각적 바(Bar)로 표현하여 볼륨 완급 조절.
        
    - **시즌/회차 계획표:** 회차별 주요 사건, **주요 복선(Foreshadowing) 입력 및 회수 체크박스**, 아이디어 메모 공간.
        
    - **인과관계 씬 디자이너:** 특정 사건 생성 시 '사건 상세 내용', '관련 인물', '공간 배경', '사건 목적', '사건의 결과'를 입력하고, 이 사건과 인과관계로 연결되는 '이어지는 사건'을 링크로 연결(클릭 시 해당 사건으로 이동).
        

### Tab 6. 워크스페이스 및 본문 에디터 (Workspace)

실제 집필이 이루어지는 공간으로, 타 탭의 데이터들과 유기적으로 상호작용하는 중심 공간.

- **UI/UX:** 3분할 레이아웃 (좌: 회차 리스트 내비게이션 / 중앙: 집필 에디터 / 우: 플로팅 사이드바).
    
- **상세 기능:**
    
    - **플로팅 사이드바 위젯:** 본문을 쓰면서 언제든지 [Tab 2. 스크랩 보드]의 포스트잇이나 [Tab 4. 캐릭터 DB]의 설정 값을 팝업/사이드 패널로 호출 가능.
        
    - **드롭앤드롭 데이터 귀속:** 스크랩 보드의 포스트잇 카드를 드래그하여 특정 캐릭터 프로필 영역이나 세계관 토글 안으로 드롭하면, 해당 설정의 '참고 데이터'로 자동 복사·귀속됨.
        

## 3. 디자이너를 위한 가이드 (UI/UX Design Guide)

1. **인터페이스 스타일:** 미니멀리즘 기반의 툴(Tool) 인상을 주어야 함. 화려한 그래픽 요소보다는 텍스트와 카드의 구분이 명확한 라인(Line) 중심의 UI.
    
2. **무한 캔버스 뷰:** 스크랩 보드는 마우스 휠 줌인/아웃, 스페이스바 누르고 드래그(Pan) 기능이 직관적으로 인지되도록 커서 모양 및 안내 툴팁 디자인 필요.
    
3. **사이드 패널 시스템:** 에디터 화면을 침범하지 않으면서 설정을 곁눈질할 수 있도록, 우측 사이드바는 접이식(Collapsible) 슬라이딩 애니메이션 적용.
    
4. **색상 전략:** 다크 모드(Slate Gray 계열 배경, Off-White 텍스트)를 기본으로 하되, 포스트잇 스크랩 카드는 파스텔톤(Muted)의 노랑, 분홍, 초록 등을 사용하여 눈에 확 띄면서도 다크 배경과 이질감이 없도록 조율.
    

## 4. 개발자를 위한 가이드 (Technical & Data Guide)

### 데이터 구조 (Entity-Relationship Diagram 힌트)

- **Workspace (1) : (N) Episode / Category**
    
- **Episode (1) : (N) Scene (Event)**
    
- **Scene (N) : (M) Character** (인물 등장표 및 씬 디자이너 연동)
    
- **ScrapCard (N) : (M) Tags / Characters / WorldBuilding** (스크랩 카드가 특정 캐릭터나 세계관 설정에 링크 및 귀속될 수 있는 다대다 구조)
    

### 핵심 기술 요구사항

1. **Open Graph Scraper:** 스크랩 보드에 URL 붙여넣기 시 백엔드 혹은 프론트엔드 프록시를 통해 `og:title`, `og:image`, `og:description`을 레이턴시 없이 파싱해야 함.
    
2. **Canvas State Management:** 무한 캔버스 내 포스트잇 카드의 X, Y 좌표값 및 Zoom Scale 상태가 실시간으로 DB에 저장 및 동기화되어야 함.
    
3. **Real-time Filtering:** 스크랩 보드의 키워드 검색 및 태그 필터링은 클라이언트 사이드에서 즉각적으로 반응하도록 메모리 내 필터 최적화(or 디바운싱 적용).
    
4. **Rich Text Editor:** 본문 에디터 및 시놉시스 에디터는 마크다운(Markdown) 단축키를 지원하는 경량 에디터 라이브러리(예: TipTap, Draft.js 등) 커스텀 활용.
