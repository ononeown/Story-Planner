## 1. 개요

- **무엇:** 장르소설(웹소설 등) 작가용 스토리 빌딩 웹 서비스. 정형 템플릿 + 무한 캔버스 스크랩을 결합한 통합 집필 환경.
- **기획서:** [`Story Planner Web.md`](Story%20Planner%20Web.md)
- **저장소:** GitHub `ononeown/Story-Planner` — **작업은 `dev` 브랜치**, 안정본은 `main`.
- **배포:** Vercel (환경변수 연결됨). DB/인증은 Supabase.
- **현 상태:** 8개 탭 + 인증 + 작품관리 + 내보내기 + AI 보조까지 핵심 기능 구현 완료. 일부 고급 기능 미구현(아래 6장).

## 2. 기술 스택

| 영역 | 사용 |
|---|---|
| 빌드/UI | Vite + React 18 + TypeScript |
| 스타일 | Tailwind CSS v4 (라이트 테마, Apple HIG 참고), Pretendard 폰트(CDN) |
| 라우팅 | react-router-dom v6 |
| 서버상태 | @tanstack/react-query (CRUD + 낙관적 업데이트) |
| DB/인증/스토리지 | Supabase (Postgres + Auth + RLS) |
| 무한 캔버스 | @xyflow/react (React Flow) — 스크랩 보드 |
| 드래그 정렬 | @dnd-kit/core, sortable — 인물/회차 |
| 에디터 | TipTap (StarterKit) — 워크스페이스 본문 |
| 내보내기 | jszip |
| AI | Gemini **웹** 연동(클립보드 복사 + 탭 열기). API 직접호출 코드는 보존하되 현재 미사용 |

> ⚠️ `zustand` 가 의존성에 있지만 **현재 코드에서 사용하지 않음**(초기 계획 잔재). 제거하거나 전역상태에 활용 가능.

## 3. 프로젝트 구조 / 구현 방식

```
src/
├─ App.tsx                  # 인증→작품 게이트 + 라우터(작품 전환 시 key=project.id 로 remount)
├─ config/tabs.ts           # 사이드바 탭 정의 (단일 소스)
├─ lib/                     # supabase, queryClient, cn, useDebouncedCallback, useBoxSelection, gemini
├─ components/
│  ├─ layout/               # AppLayout, TabNav, PageHeader, TabPlaceholder
│  └─ ui/                   # Button, Input, Textarea, Card, Field, Spinner, Segmented,
│                           #   SaveStatus, EmptyState, BulkActionBar, icons
├─ features/<도메인>/       # 탭별 기능 응집 (hook + 컴포넌트)
│  ├─ auth/  projects/  synopsis/  scrap-board/  worldbuilding/
│  ├─ characters/  timeline/  episodes-digest/  fragments/  workspace/
│  ├─ export/  ai/
└─ types/database.ts        # DB 엔티티 타입(스키마와 1:1)
```

**핵심 패턴**
- **features/<도메인> 응집:** 각 탭의 훅·컴포넌트를 한 폴더에. 탭 간 간섭 없이 성장.
- **데이터 훅:** `useXxx(workspaceId)` 형태로 react-query `list/create/update/remove` 제공. `update` 는 대부분 **낙관적 업데이트**(onMutate 캐시 갱신 → 실패 시 롤백).
- **자동 저장:** 입력은 로컬 state + `useDebouncedCallback` 으로 디바운스 후 저장. 저장 상태는 `SaveStatus`.
- **컨텍스트 2개:**
  - `AuthProvider` — Supabase 세션.
  - `ProjectProvider` — "작품(Project)" = DB `workspaces` 행. 목록/현재/생성/삭제/전환(localStorage 보존). **코드상 Project = DB workspaces** (Tab "워크스페이스"는 집필 탭으로 별개 개념).
- **다중 선택:** `useBoxSelection` — 빈 영역 범위 드래그(마퀴) + Shift 범위 + Ctrl/⌘ 토글. 항목엔 `data-select-id`. (인물·회차)
- **디자인 토큰:** `src/index.css` `@theme` 의 CSS 변수(`--color-canvas/surface/ink/accent/sticky-*` 등). 색 변경은 여기서.

## 4. 탭별 구현 현황

| # | 탭(라벨) | 라우트 | 핵심 |
|---|---|---|---|
| 1 | 시놉시스 | /synopsis | 메타데이터(작품명·장르·분량) + 로그라인·기획의도 + 5단 플롯 카드 |
| 2 | 스크랩 보드 | /scrap-board | React Flow 무한캔버스 포스트잇(메모 자동높이/링크 OG), 색칩·핀, 우클릭·Delete 삭제, 키워드 검색(비매칭 dim) |
| 3 | 세계관 | /worldbuilding | 시공간/사회/정신 카테고리 아코디언, 권장필드 칩, 항목 CRUD |
| 4 | 인물 | /characters | 그리드 + 드래그 정렬 + 등장 위계(주연/조연/단역) + 다중선택 일괄삭제 + 슬라이드 상세 패널 |
| 5 | 타임라인 | /digest | 좌: 회차별 핵심 전개 스크롤 / 우: 사건·복선 트래커(세로 리스트 + 미니 트랙바) |
| 6 | 사건 개요 | /timeline | 회차 리스트(드래그/다중선택) + 씬 디자이너(상세·인물·공간·목적·결과·지속회차·인과링크) + 복선(회수) |
| 7 | 장면 조각 | /fragments | 보고싶은 장면을 조각글로(마조네리 카드, 색·복사·삭제) → 사건으로 디벨롭 |
| 8 | 워크스페이스 | /workspace | 3분할: 회차 내비 / TipTap 본문(글자·단어수) / 참고 사이드바(캐릭터·스크랩 호출) |

**전역:** 로그인/가입, 작품 전환 드롭다운(우클릭 삭제·확인창), 내보내기(.zip), AI 집필 보조(우클릭).

## 5. 데이터 모델 (요약)

`workspaces` 아래 모든 데이터가 귀속. RLS는 `owns_workspace()` 헬퍼로 "본인 작품만" 강제.

- 1:1 `synopsis`
- 1:N `scrap_cards`, `tags`, `worldbuilding`(self-tree), `characters`, `episodes`, `scenes`, `fragments`
- N:M `scrap_card_tags`, `card_links`(다형성 귀속), `scene_characters`, `scene_links`(자기참조), `episode_characters`, `character_relations`
- `episodes` 하위: `foreshadowings`, `character_arcs`

전체 정의: [`supabase/schema.sql`](supabase/schema.sql). 자세한 설명: [`docs/DATA_MODEL.md`](docs/DATA_MODEL.md).

## 6. 미구현 / 다음 작업 후보

- **인물 고급:** 관계도 매핑(`character_relations`), 인물 변화표/Arc Matrix(`character_arcs`), 인물 등장표(`episode_characters`) — **테이블만 있고 UI 없음**.
- **워크스페이스 드래그 귀속:** 스크랩/캐릭터를 본문/설정에 드롭 → `card_links` 생성. 현재는 참고 사이드바(읽기)만.
- **태그 시스템:** `tags`/`scrap_card_tags` 테이블만 있고 스크랩 보드 태그 필터 미구현.
- **AI 진짜 임베드/자동입력:** 웹앱에선 불가(7장). 브라우저 확장 또는 Electron 필요.
- **OG 이미지 캐시:** 현재 핫링크. Supabase Storage 복사 미구현.

## 7. ⚠️ 주의해야 할 점 (반드시 읽을 것)

### 7.1 Supabase 마이그레이션 — 안 하면 기능 오류
새 컬럼/테이블은 별도 SQL 실행이 필요하다. **기존 DB에 순서대로 실행:**
- `supabase/migrations/0002_characters_sort_order.sql` — 인물 정렬 저장
- `supabase/migrations/0003_scene_span.sql` — 사건 지속 회차(`scenes.end_episode_id`)
- `supabase/migrations/0004_fragments.sql` — 장면 조각 테이블

> **새 DB**를 만들 땐 최신 `supabase/schema.sql` **하나만** 실행하면 위가 모두 포함됨.

### 7.2 환경변수
- **로컬 개발은 `.env.local` 필수** (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`). **Vercel 환경변수는 배포에만 적용**되고 로컬엔 안 들어온다.
- 미설정 시 흰 화면 대신 **SetupScreen** 안내가 뜨도록 처리돼 있음(`src/lib/supabase.ts` + `App.tsx`).
- `VITE_GEMINI_API_KEY` 는 **현재 사용 안 함**(AI가 웹 연동으로 바뀌어서). API 방식으로 되돌릴 때만 필요.

### 7.3 Git 위생 — `git add -A` 주의
- 작업 디렉토리에 **무관한 폴더가 섞여 들어온 사례 2번**(`web/`, `pretext-main/`) 있었음 → `.gitignore`에 등록해 둠.
- **커밋 시 변경 파일을 명시**(`git add <경로>`)하는 걸 권장. `-A` 쓸 거면 `git status` 먼저 확인.
- 줄바꿈 LF→CRLF 경고는 Windows 환경 정상, 무시 가능.

### 7.4 AI 집필 보조의 한계 (의도된 설계)
- gemini.google.com은 **iframe 임베드 불가**(X-Frame-Options), 다른 origin 입력창 **자동 입력 불가**(동일 출처 정책). → 그래서 "우클릭 → 프롬프트 클립보드 복사 + Gemini 탭 열기 → 사용자가 Ctrl+V·Enter" 방식.
- AI 프롬프트에 **작품 설정(시놉시스·인물·회차요약)이 포함되어 외부(Gemini)로 전송**됨. 민감정보 주의.
- 역할/프롬프트는 `src/features/ai/roles.ts`, 동작은 `src/features/ai/AiAssistant.tsx`.

### 7.5 OG 스크래퍼
- URL 스크랩 자동추출(제목/이미지/설명)은 **Supabase Edge Function `og-scrape` 배포 시**에만 동작. 미배포면 URL만으로 카드 생성(폴백).
- 배포: `npx supabase functions deploy og-scrape`. (`supabase/functions/og-scrape/index.ts`)

### 7.6 동작상 알아둘 점
- **작품 전환 시 라우터가 remount**(`key=project.id`)되어 **페이지 로컬 state가 초기화**된다(의도).
- 스크랩 카드 위치는 **드래그 종료 시점에 저장**(매 픽셀 저장 안 함).
- 회차 **드래그 정렬은 `episode_no`를 1..N로 재부여**한다(표시 번호가 바뀜, 관계는 id 기준이라 안전).
- 번들 500KB 초과 경고 있음(React Flow/TipTap). 기능엔 무관, 추후 코드 스플리팅 여지.

### 7.7 배포 반영
- 현재까지 모든 작업은 **`dev` 브랜치에만** 있고 `main`엔 미반영. Vercel이 `main`을 프로덕션으로 본다면 **확인 후 `dev`→`main` 머지** 필요.

## 8. 로컬 실행

```bash
npm install
cp .env.example .env.local   # 값 입력 (Supabase / 선택적 Gemini)
npm run dev                  # http://localhost:5173
npm run build                # 타입체크 + 프로덕션 빌드
```

작업 흐름: `dev`에서 작업·커밋·푸시 → 확인 후 `main` 머지 → 배포.
